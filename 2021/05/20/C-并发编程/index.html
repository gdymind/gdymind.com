Concurrency vs. parallelism

并发Concurrency和并行parallelism的区别是。Parallelism虽然是运行多个相同的程序，但这些程序作用在不同的数据上，不需要担心数据依赖的问题；与之对应的，concurrency会访问共享的数据，因此需要特别注意。

![/img//image-20210520170228944.png)

实现Concurrency的方法

有两种方法：

  • Multithreading：使用STL中的std::thread等对象进行实现，需要用到到了线程池（thread pool）
  • Parallelism：创建多个thread的方法overhead比较大，尤其是实现一些特别简单的功能时，thread不够轻量。所以我们还可以使用parallel execution policy annotation来实现concurrency。

Parallelism

使用parallel标记来suggests to the compiler that the function be completed concurrently with other parallel functions

使用sequential来说明that the function must be completed individually

Parallel functions can significantly speed up operations because they automatically use more of the computer’s CPU resources.

While both multithreading and parallelism are helpful concepts for implementing concurrency in a C++ program, multithreading is more widely applicable due to its ability to handle complex operations.
